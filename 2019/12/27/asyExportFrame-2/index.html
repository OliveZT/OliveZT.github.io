异步导出框架介绍（二）

Posted on 2019-12-27 | Visitors:

一. 背景

在上篇博客（异步导出框架介绍（一））中我们介绍了异步导出框架的基本架构。这次想深入一些细节，讲讲伪代码的实现。

二. 伪代码实现

首先还是祭出上篇博客中的架构图：

1. ExportDataDTO

/**
 * 需要导出的数据对象
 */
@ToString
public class ExportDataDTO implements Serializable {
}

是一个空的对象，主要是为了客户端继承，用来对象解析及数据导出。后续的导出服务的操作都是依托此对象。

2. ExportDataService 

/**
 * 数据导出服务接口
 */
public interface ExportDataService {

    /**
     * 获取待导出的数据
     */
    List<ExportDataDTO> getExportDataList(Object arg);

    /**
     * 返回文件上传后的key
     */
    String buildExcelFile(List<ExportDataDTO> exportDataList);
}

ExportDataService服务接口有两个方法，一是getExportDataList方法，主要是从导出任务规则中解析出具体导出服务的请求参数，根据此请求参数可以调用客户端的本地服务获取待导出的数据；二是将getExportDataList方法中获取的待到出数据上传到CDN中，并返回CDN的文件key。

3. ExportDataServiceFactory

/**
 * 数据导出服务工厂
 */
@Service("exportDataServiceFactory")
public class ExportDataServiceFactory {

    private static Map<Integer, ExportDataService> exportDataServiceMap = new HashMap<>();

    /**
     * 根据导出类型获取数据导出服务
     */
    public ExportDataService getExportDataServiceByType(Integer type) {
        return exportDataServiceMap.get(type);
    }

    /**
     * 将数据导出服务注册入工厂中
     */
    public void registerExportDataService(Integer type, ExportDataService exportDataService) {
        exportDataServiceMap.put(type, exportDataService);
    }
}

数据导出服务工厂ExportDataServiceFactory内部维护了一个导出类型为key，数据导出服务为value的map，主要作用即是在初始化时将具体的导出服务放入工厂中，以供使用。

4. AbstractExportDataService

/**
 * 抽象数据导出服务类
 */
public abstract class AbstractExportDataService implements ExportDataService, InitializingBean {

    @Autowired
    private ExportDataServiceFactory exportDataServiceFactory;

    @Autowired
    private FileOperateService fileOperateService;

    /**
     * 导出excel的表名
     */
    protected String sheetTitle;

    /**
     * 导出excel的表头
     */
    protected String[] header;

    /**
     * 图片所在的列的index 从0开始
     * 兼容没有图片导出的情况
     */
    protected List<Integer> imageIndex;

    /**
     * 报警接收人
     * 用于导出成功后的一条消息通知
     */
    protected String alarmReceivers;

    /**
     * excel文件导出
     */
    private String exportExcelData(List<ExportDataDTO> exportDataList, String sheetTitle, List<Integer> imageIndex) {
        // 上传到CDN的文件key
        String fileKey = null;
        try {
            Boolean checkIfExistPic = CollectionUtils.isNotEmpty(imageIndex);
            // 把数据塞到excel
            List<List<byte[]>> listData = exportDataList.stream().map(exportData -> {
                // 将数据对象转化为excel的一行数据，需要客户端自行实现
                String[] excelData = convertExportDataToExcel(exportData);
                List<byte[]> rowData = new ArrayList<>();
                // 兼容图片判断逻辑
                for (int i = 0; i < excelData.length; i++) {
                    try {
                        // 将数据转化为byte类型
                        String string = excelData[i];
                        // 图片类型则进行图片下载
                        if (checkIfExistPic && imageIndex.contains(i)) {
                            rowData.add(fileOperateService.downloadFile(string));
                            continue;
                        }
                        rowData.add(string.getBytes(CHARSET_UTF8));
                    } catch (Exception e) {
                        // 异常处理
                    }

                }
                return rowData;
            }).collect(Collectors.toList());

            // 生成excel
            Workbook workbook = new HSSFWorkbook();
            ExportExcelUtil.exportExcel(workbook, 0, true, sheetTitle, header, listData, imageIndex);
            // 上传到CDN
            ByteArrayOutputStream bos = new ByteArrayOutputStream();
            workbook.write(bos);
            fileKey = ExcelFileUploaderUtil.uploadExcelFile();
            // 下载链接发送对应告警人员
            Alarm();
        } catch (Exception e) {
            // 异常处理
        }
        return fileKey;
    }

    @Override
    public String buildExcelFile(List<ExportDataDTO> exportDataList) {
        return this.exportExcelData(exportDataList, this.sheetTitle, this.imageIndex);
    }

    @Override
    public void afterPropertiesSet() throws Exception {
        exportDataServiceFactory.registerExportDataService(getExportType(), this);
        initialProperties();
    }

    /**
     * 将导出数据DTO转化为符合excel导出格式的数据
     */
    public abstract String[] convertExportDataToExcel(ExportDataDTO exportDataDTO);

    /**
     * 获取导出的类型
     */
    protected abstract Integer getExportType();

    /**
     * 初始化excel属性，如表头等
     */
    protected abstract void initialProperties();
}

抽象的AbstractExportDataService类，实现了ExportDataService及 InitializingBean接口。AbstractExportDataService中有一个抽象的getExportType方法，在客户端实现此抽象类时需要将导出类型进行初始化。initialProperties方法主要是对需要导出的Excel表格属性如表头等进行初始化。在进行bean初始化时AbstractExportDataService会在afterPropertiesSet方法中完成具体导出服务的工厂注册以及Excel表格属性的初始化。

5. AbstractNormalExportExecuteJob

/**
 * 抽象的导出定时任务
 */
public abstract class AbstractNormalExportExecuteJob extends AbstractQuartzJob {
    @Autowired
    private ExportExcelTaskQueryService exportExcelTaskQueryService;

    @Autowired
    private ExportExcelTaskOperateService exportExcelTaskOperateService;

    protected void export(Integer exportType, Class typeClass) {
        // 根据导出类型获取没有被处理的导出任务
        List<ExportTaskResDTO> exportTaskResDTOList = exportExcelTaskQueryService.getUnExecutedExportTasksWithType(exportType);
        if (CollectionUtils.isEmpty(exportTaskResDTOList)) {
            return;
        }
        exportTaskResDTOList.forEach(exportTaskResDTO -> {
            ExportTaskReqDTO exportTaskReqDTO = new ExportTaskReqDTO();
            BeanUtils.copyProperties(exportTaskResDTO, exportTaskReqDTO);
            exportExcelTaskOperateService.executeExportExcelTask(exportTaskReqDTO, typeClass);
        });
    }

    @Override
    public void execute(ShardingContext shardingContext) {
        exportExecute(shardingContext);
    }

    public abstract void exportExecute(ShardingContext shardingContext);
}

抽象的导出定时任务，主要是根据对应的导出类型，得到没有被处理完成的数据导出任务，进行导出。其中有ExportExcelTaskQueryService和ExportExcelTaskOperateService两个服务，是框架提供的远程rpc服务

6. ExportExcelTaskQueryService和 ExportExcelTaskOperateService

/**
 * 导出查询远程rpc服务
 */
public interface ExportExcelTaskQueryService {

    /**
     * 根据exportType来获取未处理的任务
     */
    List<ExportTaskResDTO> getUnExecutedExportTasksWithType(Integer exportType);

    /**
     * 根据任务id获取任务
     */
    ExportTaskResDTO getExportTasksWithTaskId(Long taskId);
}


/**
 * 导出操作远程rpc服务
 */
public interface ExportTaskOperateService {

    /**
     * 根据条件更新导出任务
     * @param exportTaskReqDTO
     * @return
     */
    Response<Boolean> updateExportTaskByConditions(ExportTaskReqDTO exportTaskReqDTO);

    /**
     * 创建导出任务
     * @param exportTaskReqDTO
     * @return
     */
    Response<Boolean> createExportTask(ExportTaskReqDTO exportTaskReqDTO);
}

框架对外提供了远程rpc接口，用来创建及更新导出任务。底层实现即是对db进行相关操作，不再赘述。

7. createExportTask

// 框架提供的远程rpc接口
@Autowired
private ExportTaskOperateService exportTaskOperateService;

@RequestMapping("client/createExportTask")
    public String createExportTask(@RequestBody ClientExportQueryDTO clientExportQueryDTO) {
        try {
            ExportTaskReqDTO exportTaskReqDTO = new ExportTaskReqDTO();
            exportTaskReqDTO.setRule(JSON.toJSONString(clientExportQueryDTO));
            // 设置导出的业务类型
            exportTaskReqDTO.setExportType(type);
            exportTaskReqDTO.setFormName("xxx");
            // 从session获取操作人ID，用于后续的权限控制
            Long userId = SessionContextHolder.getUserId();
            exportTaskReqDTO.setUserId(userId);
            Response<Boolean> response = exportTaskOperateService.createExportTask(exportTaskReqDTO);
            if (response != null && response.isSuccess()) {
                return response.getData().toString();
            }
        } catch (Exception e) {
            // 异常处理
        }
        return "fail";
    }

客户端调用远程rpc服务创建导出任务

上面只是介绍了此框架的核心逻辑，对于一些涉及到一些基础工具如CDN文件上传及Excel下载的代码并没有贴出来，这些基础代码可以去网上进行下载，反正也没啥大的区别～

三. 关于后续优化

这周有同学过来和我对接异步下载框架。在他的项目中有超过10个地方需要使用Excel的导出功能。而在我框架设计的时候，每一个Excel的导出类型都需要一个定时任务去执行的。如果在这样的场景下，就需要建立超过10个定时任务去进行执行。虽然开发量很少，但是过多的定时任务如何合理的监管也是一个大问题；并且重复的类似代码也不够优雅（作为程序员真的要时刻保持优雅呢）。如何以更优雅更人性化的方式去处理这个问题，将是我之后的一个优化点。
