二. Java相关

1、元注解类型
1）@Retention - 标识这个注解的生存阶段。是编译期间存在，还是class中存在，还是运行期存在；
2）@Documented - 标记这些注解是否包含在用户文档中；
3）@Target - 标记这个注解应该是哪种 Java 成员（类、方法、字段、参数、构造器等）；
4）@Inherited - 标记这个注解是继承于哪个注解类(默认 注解并没有继承于任何子类)；
5）@Repeatable - Java 1.8 才加入，@Repeatable注解可以让被注解的域被相同的注解多次标注

2、其他注解
@Deprecated – @Deprecated 所标注内容，不再被建议使用。
@Override – @Override 只能标注方法，表示该方法覆盖父类中的方法。
@SuppressWarnings – @SuppressWarnings 所标注内容产生的警告，编译器会对这些警告保持静默。

3、自定义注解
和业务相关的case：
自己写了元注解，配合aop用于service异常的处理拦截。主要是为了在service层处理业务逻辑时，提供统一的异常处理逻辑，封装好特定的返回结果给上游。

4、注解与反射
注解通过反射获取。首先可以通过 Class 对象的 isAnnotationPresent() 方法判断它是否应用了某个注解

5、Java的反射机制
反射就是动态加载对象，并对对象进行解析。在运行状态中，对于任意一个类，都能够知道这个类的所有属性和方法；对于任意一个对象，都能够调用它的任意一个方法，这种动态获取信息以及动态调用对象方法的功能就称为Java反射机制。

6、反射的原理
Java语言在编译之后会生成一个class文件，反射就是通过字节码文件找到其类中的方法和属性等

7、反射中主要几个类
1）Class：类的对象；
2）Constructor：类的构造方法；
3）Field：类中的属性对象；
4）Method：类中的方法对象

8、Java8新特性
1）stream流式处理以及lamda表达式；
2）Date/Time API；
3）可重复注解；
4）接口的默认方法，接口允许非抽象方法实现

9、String、StringBuffer、StringBuilder的区别
String对象，保存在运行时常量池，String字符串的拼接会新创建出一个对象，而StringBuffer和StringBuilder做字符串拼接时并不会重新创建对象，且仅StringBuffer是线程安全的(牺牲掉了一定的执行效率)。String的循环拼接操作会被JVM自动优化为StringBuilder。

10、String.intern()方法
String.intern方法是一个native方法，会返回一个String对象。当String.intern方法调用，如果常量池中已经存在一个字符串与这个String是相等的（通过equals方法判断），那就返回常量池中的这个对象；否则就在常量池中添加这个字符串对象并返回该字符串对象的引用。
两个字符串s和t，当且仅当s.equals(t)时s.intern() == t.intern()。

jdk6中的常量池是放在 Perm 区中的，Perm区和正常的 JAVA Heap 区域是完全分开的。如果是使用引号声明的字符串都是会直接在字符串常量池中生成，而 new 出来的 String 对象是放在 JAVA Heap 区域。所以拿一个 JAVA Heap 区域的对象地址和字符串常量池的对象地址进行比较肯定是不相同的，即使调用String.intern方法也是没有任何关系的。而jdk7 的版本中，字符串常量池已经从Perm区移到正常的Java Heap区域了

11、POJO和javaBean的区别
POJO指简单普通的java对象。主要用来指代那些没有遵循特定的java对象模型，约定或者框架的对象；POJO的内在含义是指没有从任何类继承、也没有实现任何接口，更没有被其它框架侵入的java对象。POJO其实是比javabean更纯净的简单类或接口。POJO严格地遵守简单对象的概念，而一些JavaBean中往往会封装一些简单逻辑。

12、红黑树
红黑树是一种含有红黑结点并能自平衡的二叉查找树。有几个性质：1）每个节点要么是黑色，要么是红色 2）根节点是黑色 3）每个叶子节点是黑色 4）每个红色结点的两个子结点一定都是黑色 5）任意一结点到每个叶子结点的路径都包含数量相同的黑结点。
红黑树能够以O(log2(N))的时间复杂度进行搜索、插入、删除操作。此外,任何不平衡都会在3次旋转之内解决。

13、clone方法
clone方法返回一个新的对象而不是对象引用。
通过继承Cloneable接口并重写Object类中的clone方法，在重写的方法中可以调用super.clone()，然后再把浅拷贝的引用指向原型对象新的克隆体。
上述只是简单的浅拷贝，如果原型对象中含有其他对象，在克隆对象中的改动会同步反映到原型对象中；为了互不干扰，就需要深拷贝。要完成深拷贝需要将原型对象中的非基本类型也调用clone()方法。

14、创建对象的方式
1）new；
2）反射；
3）clone()方法；
4）反序列化

15、面向对象的特征
抽象、继承、封装及多态（方法的重载-编译时多态，覆盖-运行时多态）

16、抽象类与接口的区别
1）抽象类被继承，接口被实现。一个类只能继承一个类；
2）抽象类设计理念是“is-a”，接口偏重功能实现，设计理念是“like-a”；
3）接口中定义的成员变量默认是public static final，而抽象类中默认为default；
4）Java8之前，接口中不能有实现，但是抽象类中可以有非抽象的方法

17、内部类有哪些
1）静态内部类，不依赖于外部类的实例化而被实例化；
2）成员内部类；
3）局部内部类，定义在一个代码块中的类；
4）匿名内部类，没有类名的内部类，不使用关键字class、extends、implemnets，没有构造方法

18、Java自动拆箱和装箱
装箱就是自动将基本数据类型转换为包装器类型（valueOf方法）；拆箱就是自动将包装器类型转换为基本数据类型（intValue方法）。Java对于(-128,128]之间的整型数据都在常量池中缓存了对象。

19、TCP协议和UDP协议区别
1）TCP基于连接，而UDP基于无连接的；
2）对系统资源的要求：TCP较多（TCP有20个字节信息包），UDP少（UDP信息包只有8个字节）；
3）UDP程序结构较简单，传输快；
4）TCP是字节流模式，而UDP是数据报文模式；
5）TCP保证数据正确性，安全可靠，并且保证数据顺序，而UDP可能丢包，而且UDP不保证数据顺序。

20、TCP通讯过程
1）服务器端监听指定的某个端口；
2）客户端向服务端发出连接请求；
3）服务端向客户端发回接受的消息；
4）连接建立

21、TCP的三次握手（为了建立可靠的连接）
1）主机A通过向主机B 发送一个含有同步序列号的标志位的数据段给主机B，向主机B 请求建立连接；
2）主机B 收到主机A的请求后，用一个带有确认应答(ACK)和同步序列号(SYN)标志位的数据段响应主机A；
3）主机A收到这个数据段后，再发送一个确认应答，确认已收到主机B 的数据段。
为什么不是两次？由于网路阻塞，客户端第一次发送的连接没有及时到达，然后重新发送报文，然后建立通信，然后通信结束，然后第一次发送的报文到达了。然后服务端无端端的又激活了，但是客户端已经不需要在和服务端通信了。

22、TCP的四次挥手（半关闭状态）
1）当主机A完成数据传输后，将控制位FIN置1，提出停止TCP连接的请求；
2）主机B收到FIN后对其作出响应，确认这一方向上的TCP连接将关闭，将ACK置1；
3）由B 端再发出关闭请求，将FIN置1；
4）主机A对主机B的请求进行确认，将ACK置1，双方向的关闭结束。

23、OSI七层协议和TCP/IP四层协议
七层：1）物理层；2）数据链路层；3）网络层；4）传输层；5）会话层；6）表示层；7）应用层。TCP处于传输层.
四层：1）网络接口层；2）网络层；3）传输层；4）应用层

24、NIO实现原理
NIO通过selector、channel和buffer来实现非阻塞的io操作。实现时，把需要处理的channel的io事件（比如connect、read、write等）注册给selector。selector会对所有注册的channel进行轮询访问，一旦轮询到一个channel有注册事件发生，就通过回传selectionKey的方式通知开发人员对channel进行读或写操作。buffer用来保存数据。

25、通过JDBC访问数据库步骤
1）加载JDBC驱动，并注册到DriverManage中；
2）建立数据库连接，取得Connection对象；
3）建立Statement对象；
4）执行SQL；
5）返回结果集ResultSet对象；
6）依次关闭各种资源对象，释放资源

26、servlet生命周期
servlet生命周期由容器控制，可以分为加载、创建、初始化（init()方法）、处理请求（service()方法，一个请求一个线程）和卸载（destroy()方法）。

27、forward和redirect的区别
forward是服务器内部重定向，浏览器不展示跳转后的地址，重定向过程用的是同一个request；redirect是客户端重定向，浏览器地址会变，效率低于forward。

28、cookie和session的区别
1）cookie数据存放在本地浏览器，而session存放在远程服务器；
2）cookie不够安全，session数据存放在服务器，较安全；
3）cookie性能更高；
4）cookie保存数据的大小和数量会被浏览器所限制。

三. 其他问题

2、什么是布隆过滤器
布隆过滤器是一个 bit 向量或者说 bit 数组。布隆过滤器可以用于检索一个元素是否在一个集合中。它的优点是空间效率和查询时间都比一般的算法要好的多，缺点是有一定的误识别率和删除困难；
如果我们要映射一个值到布隆过滤器中，我们需要使用多个不同的哈希函数生成多个哈希值，并对每个生成的哈希值指向的 bit 位置 1。布隆过滤器误报的原理也正是因此：随着增加的值越来越多，被置为 1 的 bit 位也会越来越多，有可能某个key并没有出现过，但是其hash函数返回的位上已经因为其他key被占用了，因此被误认为存在。所以选择哈希函数的个数以及布隆过滤器长度就显的比较重要了。

3、布隆过滤器应用case
1）查某个邮箱是否是黑名单邮箱；
2）快速判断某个key是否存在

5、100G 数据，只有 100M 内存，怎么排序
1）100G 数据，按照 100M 内存拆分，然后排序有序的数据，然后写入到 file1,file2…file100；
2）多路归并排序即可：从file1…file100中取出第一个数（最小的）。所有的初始指针都是第一行，找到最小的数写入文件，并将对应的文件指针+1

6、1G大小的一个文件中找出出现频率最高的100个数，内存限制大小是1M
1）分治法，先hash映射把大文件分成很多个小文件。具体操作如下：读文件中，对于每个词x，取hash(x)%5000，然后按照该值存到5000个小文件(记为f0,f1,…,f4999)中，这样每个文件大概是200k左右（每个相同的词一定被映射到了同一文件中）；
2）对于每个文件fi，都用hash_map做词和出现频率的统计，取出频率大的前100个词（怎么取？topK问题-堆排序：建立一个100个节点的最小堆），把这100个词和出现频率再单独存入一个文件；
3）根据上述处理，我们又得到了5000个文件，归并文件取出top100（Top K问题，比较最大的前100个频数，复杂度为 NlogK）

7、bit-map位图法排序
位图法基本原理是使用位数组来表示某些元素是否存在，例如从8位号码中查重复号码，还可以用于排序。
比如排序0~15内的以下元素{1，4，7，9，15}，那就开辟一个16位的位数组，在出现的数字位上置”1“。再遍历就完成了排序。
case2: 查找一堆电话号码文件中不同号码的个数。

8、一个文件中有9亿条不重复的9位整数，对这些数字排序
1）分治法：将9亿数据hash成100份，然后在每个文件中依次搜索0～5000w，5000w01～1亿，最后将排序结果放入文件；
